GitHub Pages 发布操作指南

这份文档说明 linkserver233 的 GitHub Pages 文档站如何配置、触发、验证和排障。目标是让你在仓库维护、发版和日常文档更新时，不需要再反复翻 workflow 细节。

发布对象

当前 GitHub Pages 站点发布的是以下内容：

  • docs/ 目录下的所有静态文件
  • 根目录的 AGENTS.md
  • 根目录的 llms.txt

公开地址：

  • 站点首页：https://neko233-com.github.io/linkserver233/
  • 本页：https://neko233-com.github.io/linkserver233/github-pages.html

一次性配置

  1. 进入仓库 Settings。
  2. 打开 Pages。
  3. 在 Build and deployment 里选择 Source = GitHub Actions。
  4. 确认默认分支是 main。
  5. 确认仓库没有禁用 GitHub Actions 的 Pages 部署权限。
当前仓库已经配置成 GitHub Actions 模式，所以正常情况下不需要重复做这一步，除非仓库设置被人改回去了。

workflow 触发条件

触发方式说明
push 到 main当改动命中 docs/**、AGENTS.md、llms.txt 或 .github/workflows/pages.yml 时自动触发。
workflow_dispatch可以在 GitHub Actions 页面手动触发一次，不需要额外提交。

日常发布步骤

  1. 修改 docs/ 里的页面，或者更新 AGENTS.md / llms.txt。
  2. 本地检查文档内容是否正确。
  3. 提交并 push 到 main。
  4. 等待 pages workflow 完成。
  5. 打开站点 URL 验证页面是否已更新。
git add docs AGENTS.md llms.txt
git commit -m "docs: update GitHub Pages content"
git push origin main

本地模拟 Pages 装配

Pages workflow 实际做的事情很简单：把 docs/* 复制到一个临时站点目录，再把 AGENTS.md 和 llms.txt 一起放进去。

mkdir _site
cp -r docs/* _site/
cp llms.txt _site/llms.txt
cp AGENTS.md _site/AGENTS.md

Windows PowerShell 等价操作：

New-Item -ItemType Directory -Force _site | Out-Null
Copy-Item docs\* _site -Recurse -Force
Copy-Item llms.txt _site\llms.txt -Force
Copy-Item AGENTS.md _site\AGENTS.md -Force

GitHub 上查看与手动触发

Web UI

  1. 打开仓库的 Actions 页面。
  2. 选择 workflow：pages。
  3. 查看最近一次 run 的日志，或者点 Run workflow 手动触发。

使用 gh CLI

# 查看 Pages 配置
gh api repos/neko233-com/linkserver233/pages

# 查看最近 workflow runs
gh run list --limit 10 --json workflowName,status,conclusion,headBranch,url

# 监视 pages workflow
gh run watch <run-id> --exit-status

验证清单

  • Workflow 结论是 success
  • 站点首页可访问：/
  • 新增页面可访问，例如本页：/github-pages.html
  • AGENTS.md 和 llms.txt 在站点根目录可访问

常见问题

1. push 了但 pages workflow 没触发

通常是因为这次提交没有改到触发路径。只有 docs/**、AGENTS.md、llms.txt、.github/workflows/pages.yml 这些路径命中时才会自动触发。

2. workflow 成功了但页面还是旧的

先硬刷新，再等几十秒到几分钟的 CDN 更新。如果还是不对，检查本次 run 里实际打包进 _site 的文件。

3. 页面 404

优先检查文件名是否真的在 docs/ 下，以及链接路径是否和目标文件名一致。对于单文件页面，URL 就是文件名，例如 docs/github-pages.html 对应 /github-pages.html。

4. Pages 配置丢了

重新进入仓库 Settings → Pages，把 Source 切回 GitHub Actions。

和 release 的关系

GitHub Pages 文档发布和二进制 release 是两条独立流水线。更新文档不需要打 tag；只有发版时才需要推送 v* 标签触发 release workflow。
