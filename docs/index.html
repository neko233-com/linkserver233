linkserver233

A single-binary, self-hosted link server. Short links, custom paths, expiration, one-time links, password protection — with a clean JSON API built for agents.

Go 1.26 Zero runtime dependencies Cross-platform Agent-first

Install

macOS / Linux:

curl -fsSL https://raw.githubusercontent.com/neko233-com/linkserver233/main/scripts/install.sh | bash

Windows (PowerShell):

irm https://raw.githubusercontent.com/neko233-com/linkserver233/main/scripts/install.ps1 | iex

Go toolchain:

go install github.com/neko233-com/linkserver233/cmd/linkserver233@latest

Features

Short links

Auto-generated collision-resistant short codes.

Custom paths

Multi-segment vanity URLs like docs/latest.

Expiration

Relative (30d, 24h) or absolute RFC3339 TTLs, enforced by default.

One-time links

Atomic click budgets — burn-after-reading URLs.

Password protection

PBKDF2-hashed secrets with an agent-friendly bypass.

SSRF protection

Private / internal redirect targets are rejected.

Rate limiting

Per-client token bucket guards every endpoint.

Analytics

Click counts, last-visited timestamps, and aggregate stats.

Tags & search

Filter and paginate the link catalog.

Bulk import/export

Move catalogs between instances via JSON.

Auto-purge

Background janitor removes expired links.

Agent-first

/agent, /llms.txt, and a stable JSON contract.

Quick start

# Run the server (every link expires in 30 days by default)
linkserver233 --addr :8080 --base-url https://go.example.com

# Shorten a long URL
curl -X POST http://127.0.0.1:8080/api/v1/links \
  -H "Content-Type: application/json" \
  -d '{"target_url":"https://example.com/some/really/long/url"}'

# One-time, expiring, password-protected vanity link
curl -X POST http://127.0.0.1:8080/api/v1/links \
  -H "Content-Type: application/json" \
  -d '{"path":"launch","target_url":"https://example.com/launch","expires_in":"24h","max_clicks":1,"password":"s3cret"}'

Security & time limits

Links expire by default (--default-ttl 30d). Use --max-ttl to cap lifetimes, --require-expiry to forbid permanent links, and --admin-token to require a Bearer token on every /api/v1/* call. Redirect targets pointing at loopback, private, link-local, or cloud-metadata addresses are rejected to mitigate SSRF / open-redirect abuse.

For agents

linkserver233 ships an agent-first contract. Agents should read AGENTS.md or fetch llms.txt, then call the JSON API directly. Every link object always includes short_url, status, expires_at, and remaining_clicks.
